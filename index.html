Intro

Hi! As I wrap up the final semester of my B.S. in Computer Science, I want to take some time to reflect on this final capstone project and the program as a whole. I came into this program after graduating from a coding bootcamp and working for two years as a frontend developer. With that experience, I was concerned that getting a CS degree might be one big rehash of things I already knew, but the reason I stuck with it is that I started to realize I was getting the best of both worlds. The coding bootcamp got me comfortable with coding and taught me how to do a very specific job well enough to get hired, but the Computer Science program has given me a breadth of exposure and knowledge that would have been very difficult to get any other way. Taking classes on the Software Development Lifecycle taught me the fundamentals and methodology behind the agile workflows that I used to collaborate and stay organized with my coworkers in my day-to-day job but never really had explained to me. Classes on software security exposed a lot of the gaps surrounding common vulnerabilities, coding best practices, and security tooling that my bootcamp left because there wasn’t time to cover these topics in depth. Both of these subject areas also gave me more exposure to the business side of software development and provided me opportunities to practice communicating technical information to a wide variety of stakeholders, including users and non-technical managers. The biggest benefit that I’ve received from this program is realizing that my software engineering experience was very limited. Previously, I could code in a few very web-development-specific languages to solve a few very web-development-specific problems, but now I have so many more tools to solve so many more types of problems. I’ve built and optimized neural networks with Python, implemented more advanced data structures and weighed the efficiency and situational tradeoffs of their usage, reverse engineered binary files into assembly language and C++, and much more. I have already seen how all of these experiences have helped me be more confident and effective in my current job as a web developer. The broad exposure to so many technologies has made me a more flexible and adaptable programmer, which I think is essential in a field where no two companies will use the exact same tech stack and the technology is constantly evolving.

Project

To demonstrate some of the most relevant skills I’ve gained in this program, I chose to build a full stack web application for my capstone that I think shows my abilities in three key categories: software design and engineering, algorithms and data structures, and databases. Specifically, I wanted to show programming flexibility by porting an existing project to a completely different tech stack. The original project was an Android inventory management app that I created as the final project for my Mobile Architecture and Programming class using Java and SQLite. The updated project is a full stack web application that leverages MongoDB, Express, React, and Node (the MERN stack) to add a significant amount of functionality, security, and usability to the app. Each of the three key categories mentioned above is covered in its own section below where I give more specifics about how I improved the project.
